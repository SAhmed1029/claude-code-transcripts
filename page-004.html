    25→## Usage
    26→
    27→This tool converts Claude Code session files into browseable multi-page HTML transcripts.
    28→
    29→There are four commands available:
    30→
    31→- `local` (default) - select from local Claude Code sessions stored in `~/.claude/projects`
    32→- `web` - select from web sessions via the Claude API
    33→- `json` - convert a specific JSON or JSONL session file
    34→- `all` - convert all local sessions to a browsable HTML archive
    35→
    36→The quickest way to view a recent local session:
    37→
    38→```bash
    39→claude-code-transcripts
    40→```
    41→
    42→This shows an interactive picker to select a session, generates HTML, and opens it in your default browser.
    43→
    44→### Output options
    45→
    46→All commands support these options:
    47→
    48→- `-o, --output DIRECTORY` - output directory (default: writes to temp dir and opens browser)
    49→- `-a, --output-auto` - auto-name output subdirectory based on session ID or filename
    50→- `--repo OWNER/NAME` - GitHub repo for commit links (auto-detected if not specified). For `web` command, also filters the session list.
    51→- `--open` - open the generated `index.html` in your default browser (default if no `-o` specified)
    52→- `--gist` - upload the generated HTML files to a GitHub Gist and output a preview URL
    53→- `--json` - include the original session file in the output directory
    54→
    55→The generated output includes:
    56→- `index.html` - an index page with a timeline of prompts and commits
    57→- `page-001.html`, `page-002.html`, etc. - paginated transcript pages
    58→
    59→### Local sessions
    60→
    61→Local Claude Code sessions are stored as JSONL files in `~/.claude/projects`. Run with no arguments to select from recent sessions:
    62→
    63→```bash
    64→claude-code-transcripts
    65→# or explicitly:
    66→claude-code-transcripts local
    67→```
    68→
    69→Use `--limit` to control how many sessions are shown (default: 10):
    70→
    71→```bash
    72→claude-code-transcripts local --limit 20
    73→```
    74→
    75→### Web sessions
    76→
    77→Import sessions directly from the Claude API:
    78→
    79→```bash
    80→# Interactive session picker
    81→claude-code-transcripts web
    82→
    83→# Import a specific session by ID
    84→claude-code-transcripts web SESSION_ID
    85→
    86→# Import and publish to gist
    87→claude-code-transcripts web SESSION_ID --gist
    88→```
    89→
    90→The session picker displays sessions grouped by their associated GitHub repository:
    91→
    92→```
    93→simonw/datasette              2025-01-15T10:30:00  Fix the bug in query parser
    94→simonw/llm                    2025-01-14T09:00:00  Add streaming support
    95→(no repo)                     2025-01-13T14:22:00  General coding session
    96→```
    97→
    98→Use `--repo` to filter the session list to a specific repository:
    99→
   100→```bash
   101→claude-code-transcripts web --repo simonw/datasette
   102→```
   103→
   104→On macOS, API credentials are automatically retrieved from your keychain (requires being logged into Claude Code). On other platforms, provide `--token` and `--org-uuid` manually.
   105→
   106→### Publishing to GitHub Gist
   107→
   108→Use the `--gist` option to automatically upload your transcript to a GitHub Gist and get a shareable preview URL:
   109→
   110→```bash
   111→claude-code-transcripts --gist
   112→claude-code-transcripts web --gist
   113→claude-code-transcripts json session.json --gist
   114→```
   115→
   116→This will output something like:
   117→```
   118→Gist: https://gist.github.com/username/abc123def456
   119→Preview: https://gisthost.github.io/?abc123def456/index.html
   120→Files: /var/folders/.../session-id
   121→```
   122→
   123→The preview URL uses [gisthost.github.io](https://gisthost.github.io/) to render your HTML gist. The tool automatically injects JavaScript to fix relative links when served through gisthost.
   124→
   125→Combine with `-o` to keep a local copy:
   126→
   127→```bash
   128→claude-code-transcripts json session.json -o ./my-transcript --gist
   129→```
   130→
   131→**Requirements:** The `--gist` option requires the [GitHub CLI](https://cli.github.com/) (`gh`) to be installed and authenticated (`gh auth login`).
   132→
   133→### Auto-naming output directories
   134→
   135→Use `-a/--output-auto` to automatically create a subdirectory named after the session:
   136→
   137→```bash
   138→# Creates ./session_ABC123/ subdirectory
   139→claude-code-transcripts web SESSION_ABC123 -a
   140→
   141→# Creates ./transcripts/session_ABC123/ subdirectory
   142→claude-code-transcripts web SESSION_ABC123 -o ./transcripts -a
   143→```
   144→
   145→### Including the source file
   146→
   147→Use the `--json` option to include the original session file in the output directory:
   148→
   149→```bash
   150→claude-code-transcripts json session.json -o ./my-transcript --json
   151→```
   152→
   153→This will output:
   154→```
   155→JSON: ./my-transcript/session_ABC.json (245.3 KB)
   156→```
   157→
   158→This is useful for archiving the source data alongside the HTML output.
   159→
   160→### Converting from JSON/JSONL files
   161→
   162→Convert a specific session file directly:
   163→
   164→```bash
   165→claude-code-transcripts json session.json -o output-directory/
   166→claude-code-transcripts json session.jsonl --open
   167→```
   168→This works with both JSONL files in the `~/.claude/projects/` folder and JSON session files extracted from Claude Code for web.
   169→
   170→The `json` command can take a URL to a JSON or JSONL file as an alternative to a path on disk.
   171→
   172→### Converting all sessions
   173→
   174→Convert all your local Claude Code sessions to a browsable HTML archive:
   175→
   176→```bash
   177→claude-code-transcripts all
   178→```
   179→
   180→This creates a directory structure with:
   181→- A master index listing all projects
   182→- Per-project pages listing sessions
   183→- Individual session transcripts
   184→
   185→Options:
   186→
   187→- `-s, --source DIRECTORY` - source directory (default: `~/.claude/projects`)
   188→- `-o, --output DIRECTORY` - output directory (default: `./claude-archive`)
   189→- `--include-agents` - include agent session files (excluded by default)
   190→- `--dry-run` - show what would be converted without creating files
   191→- `--open` - open the generated archive in your default browser
   192→- `-q, --quiet` - suppress all output except errors
   193→
   194→Examples:
   195→
   196→```bash
   197→# Preview what would be converted
   198→claude-code-transcripts all --dry-run
   199→
   200→# Convert all sessions and open in browser
   201→claude-code-transcripts all --open
   202→

   204→claude-code-transcripts all -o ./my-archive
   205→
   206→# Include agent sessions
   207→claude-code-transcripts all --include-agents
   208→```
